# Code Conventions
- **Principles:** Adhere to DRY, KISS, and SOLID.
- **Typing:** Strict TypeScript only. No `any`. Use Interfaces for all data models.
- **Naming:** camelCase for variables/functions, PascalCase for classes/types. Be descriptive.
- **Error Handling:** Use functional error patterns (Result/Either) or explicit try-catches. No silent failures.
- **Documentation:** Use JSDoc for complex logic only. Code must be self-documenting.
- **Refactoring:** Leave the codebase cleaner than you found it.
